# AWS 部署檢查清單

使用此檢查清單確保所有步驟都已完成。

## 📋 準備階段

### AWS 帳號設定
- [ ] 建立 AWS 帳號
- [ ] 建立 IAM 使用者（非 root）
- [ ] 設定 MFA（多因素驗證）
- [ ] 安裝 AWS CLI（可選，方便管理）

### 專案準備
- [ ] 確認 `.gitignore` 包含 `.env*`
- [ ] 確認 `package.json` 有 `build` 和 `start` 腳本
- [ ] 確認 Prisma schema 是最新的
- [ ] 測試本地環境可以正常運行

---

## 🗄️ 資料庫設定（RDS）

### 建立 RDS 執行個體
- [ ] 選擇 MySQL 引擎和版本
- [ ] 設定執行個體類別
- [ ] 設定資料庫名稱和使用者
- [ ] 設定強密碼並記錄
- [ ] 設定網路和安全群組
- [ ] 啟用自動備份
- [ ] 等待執行個體建立完成

### 取得連線資訊
- [ ] 記錄 RDS 端點
- [ ] 記錄連接埠（預設 3306）
- [ ] 記錄資料庫名稱
- [ ] 記錄使用者名稱和密碼

### 測試連線
- [ ] 使用 MySQL 客戶端測試連線
- [ ] 或使用 MySQL Workbench/DBeaver 測試

### 資料庫遷移
- [ ] 更新 `.env` 中的 `DATABASE_URL`
- [ ] 執行 `npx prisma generate`
- [ ] 執行 `npx prisma db push` 或遷移
- [ ] 匯入現有資料（如果需要）
- [ ] 使用 `npx prisma studio` 驗證資料

---

## 📦 S3 設定

### 建立 S3 Bucket
- [ ] 選擇唯一的 bucket 名稱
- [ ] 選擇與 RDS 相同的區域
- [ ] 設定公開存取權限
- [ ] 啟用版本控制（可選）
- [ ] 啟用加密

### 設定 CORS（如果需要）
- [ ] 新增 CORS 設定
- [ ] 設定允許的來源域名

### IAM 使用者設定
- [ ] 建立 IAM 使用者（用於應用存取）
- [ ] 建立存取金鑰（Access Key）
- [ ] 記錄 Access Key ID
- [ ] 記錄 Secret Access Key
- [ ] 設定適當的權限政策

### 測試 S3
- [ ] 使用 AWS CLI 或 Console 測試上傳
- [ ] 測試下載檔案

---

## ⚙️ 環境變數設定

### 建立環境變數檔案
- [ ] 複製 `.env.aws.example` 為 `.env.aws`
- [ ] 填入所有必要的環境變數
- [ ] 確認 JWT_SECRET 是強密碼
- [ ] 確認 DATABASE_URL 正確
- [ ] 確認 S3 設定正確

### 部署平台環境變數
- [ ] 在 Vercel/Amplify 設定環境變數
- [ ] 或使用 AWS Secrets Manager（進階）

---

## 🚀 應用部署

### 選擇部署平台
- [ ] **選項 A：Vercel**
  - [ ] 連接 GitHub/GitLab
  - [ ] 匯入專案
  - [ ] 設定環境變數
  - [ ] 部署並驗證

- [ ] **選項 B：AWS Amplify**
  - [ ] 建立 Amplify App
  - [ ] 連接 Git 儲存庫
  - [ ] 建立 `amplify.yml`
  - [ ] 設定環境變數
  - [ ] 部署並驗證

- [ ] **選項 C：EC2/Docker**
  - [ ] 建立 EC2 執行個體
  - [ ] 安裝 Docker
  - [ ] 建立 Dockerfile
  - [ ] 建立 docker-compose.yml
  - [ ] 部署應用

### 更新設定檔
- [ ] 更新 `next.config.ts` 的圖片域名
- [ ] 更新 CORS 設定（如果需要）

---

## 🔐 安全設定

### RDS 安全
- [ ] 確認安全群組只允許必要 IP
- [ ] 確認資料庫密碼強度
- [ ] 確認自動備份已啟用

### S3 安全
- [ ] 確認 bucket 不公開（除非需要）
- [ ] 確認 IAM 權限最小化
- [ ] 確認啟用加密

### 應用安全
- [ ] 確認環境變數不提交到 Git
- [ ] 確認使用 HTTPS
- [ ] 確認 JWT_SECRET 是強密碼
